    多线程间通信

    从6个场景介绍线程间通信。

    • 场景一：两个线程交替执行
    • 场景二：等A执行完，再执行B
    • 场景三：先A打印1，再打印B 1，2，3，最后再打印A 2，3
    • 场景四：A B C三个线程先同步执行，都执行完后，再执行线程D
    • 场景五：A B C 各自开始准备，都准备完，再一起运行
    • 场景六：子线程完成任务后，把结果回传给主线程

    场景一：两个线程交替执行

        public static void case1() {
        Thread threadA = new Thread(new Runnable() {
            @Override
            public void run() {
                printTask("A");
            }
        });

        Thread threadB = new Thread(new Runnable() {
            @Override
            public void run() {
                printTask("B");
            }
        });

        threadA.start();
        threadB.start();
    }

    public static void printTask(String thread) {
        int i=0;
        while (i++ < 3) {
            try {
                Thread.sleep(2*1000); //每次任务都休眠，线程间不会互相影响，所以可以交替执行
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            println(thread + " printTask: " + i);
        }
    }

    public static void println(String msg) {
        System.out.println(msg);
    }

输出：

A printTask: 1
B printTask: 1
B printTask: 2
A printTask: 2
B printTask: 3
A printTask: 3

    场景二：等A执行完，再执行B

    使用Thread.join()，将当前任务放到目标任务后面。

    
    public static void case2() {
        Thread threadA = new Thread(new Runnable() {
            @Override
            public void run() {
                printTask("A");
            }
        });

        Thread threadB = new Thread(new Runnable() {
            @Override
            public void run() {
                println("B is waiting A");
                try {
                    threadA.join(); // 当前线程放到A线程后面，A结束才会执行B
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                println("B start");
                printTask("B");
            }
        });

        threadA.start();
        threadB.start();
    }

输出：
B is waiting A
A printTask: 1
A printTask: 2
A printTask: 3
B start
B printTask: 1
B printTask: 2
B printTask: 3

    场景三：先A打印1，再打印B 1，2，3，最后再打印A 2，3

    使用Object.wait() / notify()

    public static void case3() {
        Object lock = new Object();

        Thread threadA = new Thread(new Runnable() {
            @Override
            public void run() {
                synchronized (lock) { // 先获得锁，才能wait
                    println("A 1");

                    try {
                        println("A 休眠");
                        lock.wait(); // 休眠当前线程
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                    println("A 被唤醒");

                    println("A 2");
                    println("A 3");
                }
            }
        });

        Thread threadB = new Thread(new Runnable() {
            @Override
            public void run() {
                synchronized (lock) {
                    println("B 1");
                    println("B 2");
                    println("B 3");

                    println("B 唤醒休眠线程");
                    lock.notify();
                }
            }
        });

        threadA.start();
        threadB.start();
    }

输出：
A 1
A 休眠
B 1
B 2
B 3
B 唤醒休眠线程
A 被唤醒
A 2
A 3

    场景四：A B C三个线程先同步执行，都执行完后，再执行线程D

    使用CountdownLatch，让一个线程等到其他线程都结束了，再执行。

    public static void case4() {
        CountDownLatch countDownLatch = new CountDownLatch(3);

        Thread threadA = new Thread(() -> {
            printTask("A");
            countDownLatch.countDown();
        });
        Thread threadB = new Thread(() -> {
            printTask("B");
            countDownLatch.countDown();
        });
        Thread threadC = new Thread(() -> {
            printTask("C");
            countDownLatch.countDown();
        });
        Thread threadD = new Thread(() -> {
            try {
                countDownLatch.await(); // 执行到此处，当前线程会wait，直到CountDownLatch的count减少(countDown)到0，才会执行后面的操作
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            printTask("D");
        });

        threadA.start();
        threadB.start();
        threadC.start();
        threadD.start();
    }

输出：
A printTask: 1
B printTask: 1
C printTask: 1
A printTask: 2
C printTask: 2
B printTask: 2
A printTask: 3
C printTask: 3
B printTask: 3
D printTask: 1 // ABC都结束了，D才开始
D printTask: 2
D printTask: 3

    场景五：A B C 各自开始准备，都准备完，再一起运行

    CountdownLatch能够倒计数，但是当count down到0时，只有一个线程可以被唤醒，所以不适合这个场景。

    CyclicBarrier，可先指定barrier个数，即同时等待的线程数，等所有线程都准备好，即都await，就可以同时执行了。

    使用CyclicBarrier，让多个线程同步等待。等待的线程数达到CyclicBarrier的阈值，多个线程就同时开始执行任务。

    public static void case5() {
        CyclicBarrier cyclicBarrier = new CyclicBarrier(3);

        for (char threadName = 'A'; threadName <= 'C'; threadName++) {
            final String name = String.valueOf(threadName);

            new Thread(() -> {
                println(name + " waiting");
                try {
                    cyclicBarrier.await(); // 当前线程准备好，开始等待其他线程
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } catch (Exception e) {
                    e.printStackTrace();
                }
                printTask(name);
            }).start();
        }
    }

输出：
A waiting
C waiting
B waiting
B printTask: 1 // ABC都waiting，才开始执行
C printTask: 1
A printTask: 1
C printTask: 2
B printTask: 2
A printTask: 2
C printTask: 3
B printTask: 3
A printTask: 3

    场景六：子线程完成任务后，把结果回传给主线程

    Runnable是没有返回值的，所以要用Callable。结合FutureTask使用。

    public static void case6() {
        Callable<Integer> callable = new Callable<Integer>() {
            @Override
            public Integer call() throws Exception {
                println("start callable");
                int sum = 0;
                for (int i=1;i<=100;i++) {
                    sum += i;
                }

                Thread.sleep(5000);
                println("end callable");
                return sum;
            }
        };

        FutureTask<Integer> task = new FutureTask<Integer>(callable);
        new Thread(task).start(); // 会立即执行Callable

        try {
            println("before task");

            // 如果Callable没执行完，UI线程会阻塞，直至Callable结束
            println("result:" + task.get()); 

            println("after task");

        } catch (InterruptedException e) {
            e.printStackTrace();
        } catch (ExecutionException e) {
            e.printStackTrace();
        }
    }

输出：
before task.get()
start callable
end callable
result:5050
after task.get() //即便Callable里5秒耗时，但是主线程也要等其结束才能往下执行

情况2：如果注释future.get()那一行，主线程就不会等待。

输出：
start callable
before task.get()
after task.get() //主线程没有等待
end callable
